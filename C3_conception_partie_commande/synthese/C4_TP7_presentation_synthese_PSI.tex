\section{Proposition d'organisation du TP par îlot : }

\frame{
\frametitle{Proposition d'organisation}

\begin{block}{Proposition d'organisation}
Pour mener le projet, il est indispensable de se répartir le travail pour arriver à terminer le travail demandé dans le temps imparti. 
\begin{itemize}
\item Le \textbf{« chef de projet »} gère le projet, conduit l'analyse structurelle sur le système, remplie le tableau des compétences, élabore la trame pour la présentation, fait le lien entre les deux responsables \textbf{"expérimentateur"} et \textbf{"modélisateur"} pour caractériser les écarts.
\item Le responsable \textbf{« expérimentateur »} réalise les expériences sur le matériel, interprète et met en forme les résultats (pour la présentation)
\item Le responsable \textbf{"modélisateur"} travaille sur la modélisation numérique du système, interprète les résultats et les met en forme pour la présentation.
\end{itemize}
\end{block}


\begin{alertblock}<2->{Mise à disposition des données}
Les \textbf{modélisations numériques} associés à chacun des TP sont données sur le dossier "transfert" dans le répertoire \textit{PSI/TP4/"nom du tp asscocié"}
\end{alertblock}
}

\section{Analyser}
\frame{
\frametitle{Analyser}

\begin{block}{Analyser les écarts}
Les travaux pratiques sont l'occasion de déterminer les \textbf{performances réelles} d'un système.
Leur analyse présente trois principaux objectifs : 
\begin{itemize}
\item Quantification des \textbf{écarts entre performances réelles et simulées} pour vérifier la validité d'un modèle de simulation qui pourra, une fois validé, être destiné à une \textit{étude paramétrique}.
\item Quantification des \textbf{écarts entre performances réelles et attendues} pour valider les \textit{critères d'un cahier des charges} généralement précisés sur un diagramme d'exigences.
\item \textbf{Identifier} le comportement pour définir un \textit{modèle physique} associé au système
\end{itemize}
\end{block}

\section{Modéliser}
\subsection{Étapes de la modélisation}

\frame{
\frametitle{Modéliser}
La \textbf{modélisation} d'un système s'appuie sur différentes étapes.
\begin{itemize}
\item Formulation et justification des \textbf{hypothèses} : 
\begin{itemize}
\item Solides indéformables ;
\item Liaisons parfaites : sans jeu, sans frottement, contacts assurés, surfaces parfaites, etc ...
\item Référentiel d'observation galiléen; 
\item Modélisation plane ou dans l'espace ;
\end{itemize}
\item Définition des \textbf{modèles de connaissances} : 
\begin{itemize}
\item Torseurs \textbf{cinétique et dynamique} ;
\item Torseurs des actions mécaniques transmissibles (\textbf{statique}) ;
\item Traduction de l'équilibre dynamique (\textbf{Principe fondamental de la dynamique})
\end{itemize}
\end{itemize}
}

\subsubsection{Expérimenter : analyse des performances réelles d'un systèmes}


\frame{
\frametitle{Méthodologie de mesure}

\begin{enumerate}
\item Identifier la grandeur physique à mesurer qui peut être définie par le cahier des charges.
\item Mettre en place un protocole expérimental : à définir et expliciter rigoureusement.
\item Effectuer la mesure avec généralement le logiciel d'acquisition dédié.
\item Analyser les données expérimentales en travaillant par zones élémentaires.
\item Vérifier la reproductibilité de la mesure en réitérant les manipulations.
\item Analyser la chaine d'acquisition que l'on peut représenter à l'aide d'une chaine d'information.
\item Éventuellement revenir sur les hypothèses émises lors de la manipulation et de la modélisation.
\end{enumerate}

}
